# Ponytail Audit — Habit Kingdom (2026-07-10)

**Mode:** full (lazy, evidence-first). Triggered by user request to "audit" the
nightly finish→graphify→audit loop. The repo's last Ponytail pass (2026-06-22)
found ~1400 lines + 6 deps to delete but **was never applied** — this run
redoes the audit on current code with a real, searchable signal.

## Method
- Regenerated the knowledge graph (see commit 30cc1ad) — AST extraction over
  51 source files, 1449 nodes, 1490 edges.
- Cross-referenced every dependency against actual `import`/`require` usage in
  app + server + test source.
- Cross-referenced build artifacts (root `*.png` / `*.xml`) against `app.json`
  and source imports.

## Findings

### 1. Dead dependencies (removed) ✅
Three runtime deps appeared **only in package.json** — zero source, test, or
config references. Confirmed safe to drop:

| Package | Why dead |
|---|---|
| `@babel/plugin-proposal-private-property-in-object` | Babel 7.21+ / RN 0.81 handles `#private` natively; not in `babel.config.js`. |
| `@stardazed/streams-text-encoding` | No import anywhere; superseded by RN 0.81 built-in streams. |
| `http-proxy-middleware` | No proxy route in `server/`; never required. |

### 2. Dead build artifacts (removed) ✅
Root-level `*.png` + `*.xml` mockups (`screenshot*.png`, `ui*.xml`,
`screenshot_wallet.png`, `ui_wallet2.xml`) — **not imported** by `app.json` or
any source. They were stray exports. APKs kept (real deliverables).

### 3. Stale graph artifact (removed) ✅
`graphify-out/graph.html` (482 KB) — a binary visualization from the old May-30
graph the local extractor can't regenerate. Kept `graph.json` + `GRAPH_REPORT.md`.

### 4. Things that LOOKED dead but are NOT — kept ⚠️
The dependency + dead-export scans flagged 35 "unused" deps and 92 "unused"
exports. **Most are false positives** from barrel re-exports and file-based
routing:
- `react`, `react-native`, `expo-*`, `@expo/vector-icons` — consumed via `expo`
  re-exports; not directly imported.
- All `app/(tabs)/*` screens — expo-router **file-based routing** (no name
  reference needed).
- `lib/db.ts` public API (`insertProfile`, `getHabitById`, …) — called via
  dynamic storage abstraction; name not always lexically present.
- `@ungap/structured-clone` — imported in `index.js` polyfill. **Keep.**
- `fbemitter` — active sync event bus in `lib/storage.ts`. **Keep.**
- Server deps (`express`, `helmet`, `bcrypt`, `pg`, `zod`, `ws`, …) — imported
  in `server/` (excluded from the lazy source glob but real).

**Decision:** did NOT bulk-delete these. Ponytail rule — delete only when
*provably* dead. Regex name-matching is insufficient proof for re-exported /
router-resolved symbols. Listing them here for a future, per-file human review.
